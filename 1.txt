1. Members (会员表) 仅用于存储会员信息，包括其登录凭证。
字段名	数据类型	约束/备注
member_id	INT	PRIMARY KEY, 会员唯一标识
phone_number	VARCHAR(20)	UNIQUE, 会员手机号 (用于登录和查询)
password_hash	VARCHAR(255)	密码的哈希值 (针对会员网站登录)
name	VARCHAR(100)	会员姓名
member_level	VARCHAR(50)	会员等级
wallet_balance	DECIMAL(10,2) 	会员钱包余额，默认0.00 ；钱包余额等于返利余额+充值金额
rebate_balance	DECIMAL(10,2)	会员返利余额，默认0.00
status	VARCHAR(20)	会员状态 (如: '活跃', '待注销', '已注销')
2. Employees (员工表) 仅用于存储员工信息，包括其登录凭证和角色。
字段名	数据类型	约束/备注
employee_id	INT	PRIMARY KEY, 员工唯一标识 (如员工号)
username	VARCHAR(100)	UNIQUE, 员工登录账号
password_hash	VARCHAR(255)	密码的哈希值
name	VARCHAR(100)	员工姓名
role	VARCHAR(50)	员工角色 (如: '收银员', '仓库管理员', '经理')
status	VARCHAR(20)	员工状态 (如: '在职', '禁用')
3. Products (商品表) 包含商品的详细信息和库存数据，保持不变。
字段名	数据类型	约束/备注
product_id	INT	PRIMARY KEY, 商品唯一标识
product_code	VARCHAR(50)	UNIQUE, 商品条码或编号
product_name	VARCHAR(255)	商品名称
category	VARCHAR(100)	种类
specification	VARCHAR(255)	规格
purchase_price	DECIMAL(10,2)	进货价格
retail_price	DECIMAL(10,2)	零售价格
current_stock	INT	当前库存量
stock_location	VARCHAR(100)	库存位置 (简单表示)
low_stock_threshold	INT	低库存预警值
is_active	BOOLEAN	商品是否在售 (用于商品下架/删除，TRUE表示在售)
4. Orders (订单表) 记录销售订单的主信息，关联到 Members 表和 Employees 表。
字段名	数据类型	约束/备注
order_id	INT	PRIMARY KEY, 订单唯一标识
order_date	DATETIME	订单创建时间
total_amount	DECIMAL(10,2)	订单总金额
member_id	INT	FOREIGN KEY references Members(member_id), 关联会员 (若为会员购物，则关联会员；若为游客购物，则为空)
cashier_employee_id	INT	FOREIGN KEY references Employees(employee_id), 关联收银员
payment_method	VARCHAR(50)	支付方式 (如: '现金', '银行卡刷卡', '移动支付', '会员钱包扣款') 
order_status	VARCHAR(20)	订单状态 (如: '已完成', '已退款')
rebate_earned	DECIMAL(10,2)	本次订单产生的返利金额 (针对会员)
5. Order_Items (订单明细表) 记录每次订单中包含的商品详情。
字段名	数据类型	约束/备注
order_item_id	INT	PRIMARY KEY, 订单商品项唯一标识
order_id	INT	FOREIGN KEY references Orders(order_id), 关联订单
product_id	INT	FOREIGN KEY references Products(product_id), 关联商品
quantity	INT	购买数量
unit_price	DECIMAL(10,2)	购买时商品单价
item_total	DECIMAL(10,2)	该商品项总金额 (quantity * unit_price)
6. Product_Loss_Records (商品损耗记录表) 记录商品的非销售损耗，关联到 Employees 表。
字段名	数据类型	约束/备注
loss_record_id	INT	PRIMARY KEY, 损耗记录唯一标识
product_id	INT	FOREIGN KEY references Products(product_id), 关联损耗商品
quantity_lost	INT	损耗数量
loss_reason	VARCHAR(255)	损耗原因 (如: '过期', '变质', '破损', '报废')
loss_date	DATETIME	记录损耗日期和时间
recorded_by_employee_id	INT	FOREIGN KEY references Employees(employee_id), 记录者 (仓库管理员)

7. Refunds (退款记录表) 记录所有商品退款的详细信息。这是为了满足“商品退款”需求而新增的表。
字段名	数据类型	约束/备注
refund_id	INT	PRIMARY KEY, 退款唯一标识
order_id	INT	FOREIGN KEY references Orders(order_id), 关联原订单
product_id	INT	FOREIGN KEY references Products(product_id), 退款商品ID
refund_quantity	INT	退款数量
refund_amount	DECIMAL(10,2)	退款金额
refund_date	DATETIME	退款操作时间
refund_reason	VARCHAR(255)	退款原因
refund_method	VARCHAR(50)	退款方式 (如: '退回现金', '原路返回', '退回会员钱包') 
cashier_employee_id	INT	FOREIGN KEY references Employees(employee_id), 办理退款的收银员
rebate_deducted	DECIMAL(10,2)	退款扣除的返利金额 (针对会员)
8.Member_Wallet_Rebate_History (会员钱包/返利历史表) 为了满足“查看个人信息：查询...历史消费记录”和“账户充值”的需求，并保持精简，将钱包和返利的历史记录合并到一个表中。
字段名	数据类型	约束/备注
history_id	INT	PRIMARY KEY, 历史记录唯一标识
member_id	INT	FOREIGN KEY references Members(member_id), 关联会员
transaction_type	VARCHAR(50)	交易类型 (如: '钱包充值', '钱包消费', '钱包退款入账', '返利获得', '返利扣减')
amount	DECIMAL(10,2)	变动金额 (正数表示增加，负数表示减少)
transaction_date	DATETIME	交易时间
related_order_id	INT	FOREIGN KEY references Orders(order_id), 关联订单 (如果是消费/返利产生/退款)
